# تقرير إصلاح قالب طباعة لصاقة 100×80mm في CLOTEX

تاريخ التقرير: 2026-05-03  
نطاق المهمة: إصلاح قص وتشويه لصاقة أتواب الأقمشة 100×80mm فقط، بدون إعادة تصميم التطبيق أو تغيير بيانات اللصاقة.

## 1. الملخص

تم ضبط مسار طباعة لصاقة أتواب الأقمشة ليطبع على مقاس فعلي:

- العرض: `100mm`
- الارتفاع: `80mm`
- هامش الصفحة: `0`
- الهامش الداخلي الآمن: `1mm`
- مساحة المحتوى الفعلية: `98mm × 78mm`

تمت إزالة سبب القص الأساسي: كان مسار الطباعة يتعامل مع لصاقة 100×80 كصفحة 80×100 مع تدوير المحتوى `rotate(-90deg)` وقياس داخلي. هذا السلوك كان يسبب قص الأعلى/اليمين عند بعض تعريفات الطابعات الحرارية أو عند الطباعة إلى PDF.

بعد الإصلاح أصبحت وثيقة الطباعة تستخدم `@page { size: 100mm 80mm; margin: 0; }` مباشرة، بدون تدوير وبدون تكبير.

## 2. سبب القص

الملف الأساسي:

- `src/components/labels/LabelCard.tsx`

المشكلة كانت في `buildPrintDocument()`:

- عند `pageSize='label'` وكون العرض أكبر من الارتفاع، كان الكود يعتبر اللصاقة `rollThermalWide`.
- كان يضبط الصفحة كـ `80mm × 100mm`.
- كان يلف المحتوى داخل `.label-roll-wide`.
- كان يطبق:
  - `transform: rotate(-90deg)`
  - `scale(...)`

هذا الأسلوب حساس جداً لطابعات Windows وتعريفات thermal labels، لأنه يجعل حجم الصفحة الفعلي مختلفاً عن مقاس اللصاقة المطلوب، ثم يعتمد على دوران داخلي. النتيجة المتوقعة عملياً: قص عند الأعلى أو اليمين، أو هامش غير صحيح، أو تشويه في تموضع الباركود والـ QR.

## 3. الملفات المعدلة

| الملف | التعديل |
|---|---|
| `src/components/labels/LabelCard.tsx` | إزالة مسار تدوير 100×80، تثبيت `@page 100mm 80mm`، ضبط margin/padding، ضبط barcode/QR sizing |
| `electron/main.ts` | ضبط custom page size في Electron إلى 100×80 مباشرة، وإجبار PDF الخاص باللصاقات على margins none |
| `src/pages/inventory/StickerPrinting.tsx` | جعل الحجم الافتراضي في شاشة طباعة اللصاقات هو لصاقة منفصلة 100×80 بدلاً من A4 |

لم يتم تعديل بيانات اللصاقة، توليد الباركود، توليد QR، Excel import، أو صفحات وحدات الأعمال.

## 4. إصلاح CSS @page والهامش

أصبح مسار طباعة اللصاقة ينتج CSS بهذا المعنى:

```css
@page {
  size: 100mm 80mm;
  margin: 0;
}

html,
body {
  margin: 0 !important;
  padding: 0 !important;
  width: 100mm;
  min-height: 80mm;
}

.label-page {
  width: 100mm;
  height: 80mm;
  max-width: 100mm;
  max-height: 80mm;
  box-sizing: border-box;
  padding: 1mm;
  overflow: hidden !important;
}
```

النقطة المهمة: لم يعد هناك تدوير داخلي، ولم يعد هناك صفحة 80×100 للّصاقة 100×80.

## 6. ضبط QR والباركود

تم تثبيت قياسات العناصر الحساسة:

| العنصر | القياس |
|---|---|
| QR | `22mm × 22mm` |
| Barcode SVG | `64mm × 14mm` |
| Barcode wrapper | `66mm × 14mm` مع padding داخلي `1mm` كـ quiet zone |
| Barcode text | `3.2mm` مع منع الخروج خارج العرض |

الهدف من هذا الضبط:

- منع تمدد الباركود خارج حدود اللصاقة.
- الحفاظ على quiet zone للباركود.
- منع ضغط QR بشكل غير متساوٍ.
- إبقاء QR والباركود داخل مساحة 98×78mm.

## 7. سلوك الطباعة Web

مسار المتصفح ما زال يستخدم:

- `window.open()`
- `window.print()`

لكن HTML الذي يرسله المتصفح أصبح مضبوطاً على:

- `@page 100mm 80mm`
- `margin: 0`
- `.label-page` بحجم 100×80
- padding داخلي 1mm
- بدون `rotate(-90deg)`

ملاحظة: بعض المتصفحات أو تعريفات الطابعة قد تضيف هوامش إذا لم يكن حجم الورق مضبوطاً في نافذة الطباعة. لذلك يجب اختيار حجم ورق 100×80mm وتعطيل fit/shrink من تعريف الطابعة.

## 8. سلوك الطباعة Electron

الملف:

- `electron/main.ts`

تم تعديل `buildElectronPageSize()`:

- سابقاً كان يعكس 100×80 إلى 80×100 للطابعات الحرارية.
- الآن يرسل الحجم كما هو:
  - width = `100000` microns
  - height = `80000` microns

كذلك في `printToPDF()`:

- إذا كانت الصفحة `ROLL_LABEL`، يتم استخدام:
  - `margins: { marginType: 'none' }`

أما `webContents.print()` فكان يستخدم بالفعل `marginType: none` للّصاقات، وتم الحفاظ عليه.

## 9. نتائج التحقق الآلي

تم توليد HTML عينة من `buildPrintDocument()` وفحصه آلياً.

النتيجة:

| الفحص | النتيجة |
|---|---|
| يحتوي `@page { size: 100mm 80mm; margin: 0; }` | نعم |
| يحتوي margin صفر للـ html/body | نعم |
| يحتوي padding داخلي `1mm` | نعم |
| يحتوي `rotate(...)` | لا |
| يحتوي `label-roll-wide` | لا |
| Barcode مثبت على `64mm` | نعم |
| QR مثبت على `22mm` | نعم |

هذا يثبت أن HTML المرسل للطباعة لم يعد يستخدم تدوير أو قياساً يسبب القص.

## 11. اختبارات الطباعة اليدوية

لم يتم تنفيذ طباعة فعلية على طابعة حرارية من هذه الجلسة، لأن ذلك يحتاج جهاز الطابعة وتعريفها الفيزيائي.

تم التحقق برمجياً من HTML/CSS الناتج، ويجب إجراء الاختبار اليدوي التالي على جهاز الطباعة:

1. فتح شاشة طباعة لصاقات الأتواب.
2. اختيار لصاقة منفصلة 100×80mm.
3. الطباعة إلى PDF أولاً بحجم ورق 100×80mm.
4. التأكد من:
   - عدم قص الأعلى.
   - عدم قص اليمين.
   - عدم وجود هوامش بيضاء كبيرة.
   - ظهور `CLOTEX` بالكامل.
   - ظهور QR بالكامل.
   - ظهور الباركود بالكامل.
   - ظهور رسالة footer بالكامل.
5. الطباعة الفيزيائية على لصاقة 100×80mm.
6. مسح الباركود.
7. مسح QR.
8. اختبار نفس القالب من:
   - طباعة لصاقة ثوب يدوي.
   - auto-print بعد إنشاء ثوب.
   - طباعة لصاقات دفعة استيراد.

## 12. ملاحظات معايرة الطابعة

إذا استمر قص 1–2mm بعد هذا الإصلاح، فالمشكلة غالباً من تعريف الطابعة أو إعداد الورق، ويجب ضبط الآتي:

- Paper size في تعريف الطابعة = `100mm × 80mm`.
- Margins = none / بدون هوامش.
- Scale = `100%`.
- تعطيل:
  - Fit to page
  - Shrink to printable area
  - Auto rotate إذا كان يغير الاتجاه.
- معايرة gap sensor في الطابعة.
- التأكد من أن اتجاه الورق في driver يطابق CSS: عرض 100mm وارتفاع 80mm.

## 13. القيود المعروفة

- لا يمكن ضمان الطباعة الفيزيائية بنسبة 100% بدون معايرة تعريف الطابعة، لأن بعض تعريفات Windows تفرض printable area حتى مع `@page margin: 0`.
- في المتصفح، يجب اختيار حجم الورق الصحيح يدوياً إذا لم يتعرف المتصفح عليه تلقائياً.
- Electron silent print يعتمد على دعم تعريف الطابعة لـ custom page size بالميكرون.
