# تقرير: أداة سطر أوامر لتوليد مفاتيح التفعيل السحابية — CLOTEX

**التاريخ:** 2026-05-03

---

## 1. ملخص

تمت إضافة أمر **`npm run activation:generate`** يستدعي السكربت **`server/src/scripts/generateActivationKeys.ts`** لتوليد مفاتيح التفعيل بصيغة **`XXXX.XXXX.XXXX.XXXX`** وتخزين **التجزئة فقط** في PostgreSQL مع لاحقة **`key_suffix`** وسجل حدث **`KEY_GENERATED`**، دون الحاجة لتسجيل دخول ويب. تُطبع المفاتيح الخام **مرة واحدة** في الطرفية وتُنسخ اختيارياً إلى ملف محلي تحت **`server/generated/`** (المجلد مُهمل في Git).

---

## 2. لماذا كانت الأداة مطلوبة؟

عند فرض **التفعيل قبل استخدام النظام**، يصبح توليد المفاتيح من **إعدادات النظام داخل التطبيق** حلقة مفرغة: لا يدخل المسؤول قبل التفعيل، ولا يُنشَأ مفتاح قبل الدخول. الـ **CLI** يكسر هذه الحلقة بتوليد المفاتيح مباشرة من بيئة الخادم/قاعدة البيانات قبل أي تفعيل.

---

## 3. الملفات المُنشأة / المُعدَّلة

| الملف | التغيير |
|-------|---------|
| `server/src/scripts/generateActivationKeys.ts` | **جديد** — أداة CLI، تحقق من `DATABASE_URL` و`ACTIVATION_KEY_PEPPER`، استدعاء `generateActivationKeys`. |
| `server/src/services/activationService.ts` | خيار اختياري **`eventMessage`** في `generateActivationKeys` لتمييز أحداث CLI عن واجهة API. |
| `package.json` | سكربت **`activation:generate`**. |
| `docs/CLOTEX_ADMIN_CLI_ACTIVATION_KEY_GENERATOR_REPORT.md` | هذا التقرير. |

**ملاحظة:** مجلد **`server/generated/`** كان مُدرجاً مسبقاً في **`.gitignore`**.

## 4. طريقة الاستخدام

### الأمر الأساسي

```bash
npm run activation:generate -- --count=20 --plan=FULL
```

### القيم الافتراضية

- **`count`:** 20  
- **`plan`:** `FULL`  

### خيارات إضافية

| الخيار | الوصف |
|--------|--------|
| `--count` | عدد المفاتيح (1–100، يُرفض أي عدد > 100) |
| `--plan` | `LITE` أو `PRO` أو `FULL` |
| `--expiresAt` | تاريخ انتهاء اختياري (يُمرَّر للسجل كما هو) |
| `--notes` | ملاحظات تُخزَّن مع السجل |
| `--output` | مسار ملف نصي بديل؛ إن لم يُحدَّد يُنشَأ ملف تلقائي بالطابع الزمني |

### أمثلة

```bash
npm run activation:generate -- --count=3 --plan=FULL
npm run activation:generate -- --expiresAt=2027-12-31 --notes="دفعة عملاء"
npm run activation:generate -- --output=server/generated/my-keys.txt
```

### توافق Windows / npm

قد يمرّر بعض إصدارات npm على Windows الوسائط كوسائط **موضعية** (مثل `tsx … 20 FULL`). السكربت يدعم ذلك تلقائياً. بديل موثوق:

```bash
npx tsx server/src/scripts/generateActivationKeys.ts --count=20 --plan=FULL
```

أو بالشكل الموضعي المعاد من npm:

```bash
npm run activation:generate -- --count 20 --plan FULL
```

## 5. السلوك الأمني

| القاعدة | التنفيذ |
|---------|---------|
| اتصال PostgreSQL | عبر **`DATABASE_URL`** في `server/.env` (يتم التحقق عبر تحميل البيئة؛ لا يُطبع الرابط في السجلات الناجحة). |
| تجزئة المفتاح | نفس الدالة **`hashActivationKey`** مع **`ACTIVATION_KEY_PEPPER`** — إن كان فارغاً يتوقف السكربت برسالة خطأ. |
| التخزين في DB | **`key_hash`** + **`key_suffix`** فقط؛ لا يُخزَّن المفتاح الخام. |
| الطباعة | المفاتيح الخام في الطرفية **مرة واحدة** بعد النجاح؛ لا تُطبع التجزئة. |
| ملف المالك | نسخة نصية تحذيرية تحت **`server/generated/`**؛ **لا تُستخدم للتحقق المحلي** — هي أرشيف للمالك فقط. |
| صلاحية الترخيص السحابي | لم يُغيَّر منطق الخادم؛ المفاتيح صالحة عبر **سلطة التفعيل على VPS/PostgreSQL** كما في الواجهة. |

## 6. نتائج الاختبار (بيئة التطوير)

| الخطوة | النتيجة |
|---------|---------|
| `npm run server:check` | نجاح |
| `npm run build` | نجاح |
| `npm run server:migrate` | نُفِّذ لتطبيق الجداول بما فيها `activation_keys` |
| `npm run activation:generate -- --count 1 --plan LITE` | نجاح — طُبع مفتاح واحد، وملف تحت `server/generated/activation-keys-*.txt` |
| التحقق من عدم تخزين المفتاح الخام في DB | المنطق يعتمد على نفس مسار API؛ الحقل **`key_hash`** فقط في الإدراج |

**اختبارات يدوية موصى بها على الخادم:**

- التفعيل من نافذة تسجيل الدخول بأحد المفاتيح المُولَّدة.  
- محاولة تفعيل ثانية بنفس المفتاح (يُفترض الفشل وفق سياسة **`max_activations`** وما يعادل «مفتاح مستخدم» في الخدمة).

---

## 7. أين يجد المالك المفاتيح؟

1. **الطرفية** — مباشرة بعد التشغيل الناجح (النسخة الوحيدة «الساخنة»).  
2. **ملف محلي** — المسار يُعلَن في السطر الأخير من السجل، عادة:  
   `server/generated/activation-keys-YYYYMMDD-HHmmss.txt`  
   ما لم يُستخدم **`--output`**.
