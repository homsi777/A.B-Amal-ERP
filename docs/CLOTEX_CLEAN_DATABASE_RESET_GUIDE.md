# دليل إعادة تهيئة قاعدة البيانات (تطوير فقط)

## تحذير

هذا الإجراء **يحذف كل الجداول والبيانات** في مخطط `public` على PostgreSQL ثم يُعيد تشغيل الترحيلات والبذور الدنيا فقط (شركة، admin، عملات، مستودع MAIN، أدوار، قالب لصاقة، إلخ).

لا تستخدمه على إنتاج حقيقي إلا بعد نسخ احتياطي كامل ولسبب واضح.

## المتطلبات

- `DATABASE_URL` صحيح في `server/.env`
- المتغير **`ALLOW_DB_RESET=true`** يجب أن يكون مضبوطاً عند تشغيل الأمر (يُضبط تلقائياً في سكربت npm الموضّح أدناه).

## الأمر

من جذر المشروع:

```bash
npm run server:reset:clean
```

ما يحدث:

1. التحقق من `ALLOW_DB_RESET=true`
2. `DROP SCHEMA public CASCADE` ثم `CREATE SCHEMA public`
3. `npm run server:migrate`
4. `npm run server:seed`

## بعد الإكمال

- تسجيل الدخول: المستخدم `admin` وكلمة المرور حسب `SEED_ADMIN_PASSWORD` أو في التطوير بدون متغير: `admin123`
- الشركة: رمز `CLOTEX-MAIN`، اسم `CLOTEX`
- لا عملاء/موردين/أتواب حتى تُضاف من الواجهة أو الاستيراد

## بديل يدوي

إذا لم ترغب بتشغيل السكربت، يمكن تنفيذ نفس خطوات الحذف والإنشاء يدوياً من عميل SQL ثم `npm run server:migrate` و `npm run server:seed`.
