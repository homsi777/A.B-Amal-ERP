# تقرير تنفيذ: مركز التقارير MVP (CLOTEX)

**التاريخ:** 2026-05-03  
**النطاق:** ربط مركز التقارير بـ PostgreSQL، فلاتر، تصدير Excel، طباعة / PDF عبر المتصفح، دون إعادة تصميم واجهة المالك.

---

## 1. الملخص

تم تحويل **مركز التقارير** من واجهة «غير مفعّلة» إلى **MVP فعلي** يسحب البيانات من الخادم عبر JWT و `company_id`، مع دعم الحالات الفارغة والتصدير والطباعة. **لم يُستخدم Zustand أو localStorage كمصدر تقارير**، و**لا صفوف وهمية**. التقارير المحاسبية الثقيلة تبقى ظاهرة كـ «قريباً» برسالة مهنية دون بيانات زائفة.

**الحكم النهائي:**  
تم تحويل مركز التقارير إلى MVP فعلي مرتبط بـ PostgreSQL مع تصدير Excel وطباعة/PDF دون كسر التصميم.

---

## 2. التقارير المفعّلة (MVP)

| المجال | التقرير |
|--------|---------|
| لوحة تنفيذية | الملخص التنفيذي العام (لوحة القيادة + أعداد موسّعة) |
| مخزون | كشف أتواب المخزون، حركة الأتواب، الأتواب حسب المستودع، الأتواب حسب الخامة واللون |
| شراء/استيراد | سجل دفعات استيراد Excel، صفوف دفعة الاستيراد |
| مالية مبسطة | أرصدة الصناديق، حركة الصندوق، سجل السندات |
| أطراف | نشاط العملاء والموردين |
| طباعة | سجل مهام الطباعة |
| رواتب | ملخص الرواتب والموظفين |

**غير مطبّق في هذا الإصدار (متعمد):** دفتر الأستاذ العام، ميزان المراجعة، قائمة الدخل، الميزانية العمومية، التدفقات النقدية، فروقات العملة — تظهر كتقارير «محاسبية متقدمة» مع `ComingSoonReport`.

---

## 3. واجهات البرمجة الخلفية (Backend)

**الملفات الرئيسية:**  
`server/src/routes/reportRoutes.ts`، `server/src/services/reportService.ts`، `server/src/services/reportTypes.ts`

جميع المسارات أدناه تتطلب مصادقة وتُقيَّد بـ `company_id` من JWT.

### ملخص تنفيذي موسّع

- `GET /api/reports/dashboard-summary` — أعداد العملاء، الموردين، المستودعات، الأتواب، النشطة، التالفة/الهدر إن وُجد، دفعات الاستيراد، مهام الطباعة، الصناديق، السندات، مرتجعات، موظفو الرواتب، مسيرات الرواتب، إجمالي أمتار/أوزان تقديرية، أرصدة نقدية حسب العملة، إلخ.

### تقارير موحّدة (شكل `UnifiedReportPayload`)

- `GET /api/reports/inventory/rolls`
- `GET /api/reports/inventory/movements`
- `GET /api/reports/inventory/by-warehouse`
- `GET /api/reports/inventory/by-item-color`
- `GET /api/reports/purchases/import-batches`
- `GET /api/reports/purchases/import-rows`
- `GET /api/reports/financial/cashboxes`
- `GET /api/reports/financial/cashbox-movements`
- `GET /api/reports/financial/vouchers`
- `GET /api/reports/parties/activity`
- `GET /api/reports/printing/jobs`
- `GET /api/reports/payroll/summary`

**استجابة موحّدة:** `{ ok: true, report: { title, generatedAt, filtersApplied, columns, rows, totals?, summaryCards?, meta? } }`

**ملخصات قديمة متوافقة:** ما زالت نقاط مثل `inventory-summary`, `cashbox-summary`, `vouchers-summary`, `payroll-summary` تُستخدم في **الملخص التنفيذي** إلى جانب `dashboard-summary`.

## 4. الواجهة الأمامية (Frontend)

| المكوّن / الملف | الدور |
|-----------------|--------|
| `src/pages/reports/ReportsCenter.tsx` | تبويبات، بطاقات تقارير، فلاتر عليا، ربط مفاتيح التقارير (`selectedKey`) بمسارات API، لوحة الملخص التنفيذي |
| `src/components/reports/ReportViewer.tsx` | جدول RTL، تحميل/خطأ/فراغ، ترقيم صفحات عند وجود `meta` |
| `src/components/reports/ReportToolbar.tsx` | تحديث، Excel، طباعة/PDF، نسخ (حسب توفر البيانات) |
| `src/components/reports/ComingSoonReport.tsx` | تقارير غير الجاهزة أو المحاسبية المتقدمة |
| `src/lib/api/reportsApi.ts` | `getDashboardSummary`, `fetchUnifiedReport`, الملخصات المساعدة |
| `src/lib/reports/types.ts` | نوع الحمولة الموحّدة للعميل |
| `src/lib/reports/exportReportToExcel.ts` | تصدير Excel بـ `xlsx` |
| `src/lib/reports/printReport.ts` | HTML للطباعة + `window.print()` |

**ملاحظة تصميم:** طُلب مكوّن منفصل `ReportFilters.tsx` في المواصفات؛ الفلاتر **دُمجت في الشريط العلوي لـ ReportsCenter** للحفاظ على التخطيط المعتمد وتجنّب تكرار الشاشة — السلوك مطابق (فلاتر تؤثر على استدعاء API).

## 5. التصدير والطباعة

- **Excel:** `exportReportToExcel` — عنوان، تاريخ توليد، ملخص فلاتر، رؤوس عربية، صفوف، إجماليات وملخص إن وُجد، اسم ملف `clotex-report-<key>-YYYYMMDD-HHmm.xlsx`.
- **PDF:** يعني حالياً **طباعة المتصفح / حفظ كـ PDF** عبر `printReport` — بدون مُولّد PDF على الخادم.

---

## 6. سلوك الفلاتر

- **عام:** `dateFrom`, `dateTo`, `search` تُمرَّر للتقارير التي تدعمها في الخادم.
- **مستودع:** يُطبَّق على تقارير المخزون ذات الصلة (`warehouseId`).
- **صندوق (UUID):** لحركة الصندوق وسجل السندات (`cashboxId`).
- **دفعة استيراد:** لتقرير الصفوف — حقل UUID وزر «تحميل الصفوف» يزيد `applyNonce` ويعيد التحميل.
- **«تطبيق وحساب»:** يعيد تعيين الصفحة إلى 1 ويزيد عداد التطبيق فيُعاد جلب التقرير.

---

## 7. التقارير المعطّلة / «قريباً»

- **محاسبة متقدمة:** بطاقات `mode: 'accounting'` تعرض `ComingSoonReport` مع نص يوضح الحاجة لمحرك دفاتر — **لا رسوم ولا صفوف وهمية**.
- **تقارير أخرى غير MVP:** `mode: 'soon'` برسالة احترافية دون تعطيل تصميم البطاقات.

---

## 8. الحالة الفارغة

- الجداول بدون صفوف: رسالة افتراضية في `ReportViewer`: «لا توجد بيانات لهذا التقرير ضمن الفلاتر الحالية».
- الملخص التنفيذي: أصفار وأعداد صفرية عند قاعدة نظيفة.
- التصدير: يمكن تصدير ملف يحتوي الرؤوس فقط إذا طلب المستخدم ذلك.

---

## 9. الأمان والنطاق (شركة)

- كل استعلام SQL يفرض `company_id` للمستخدم المصادَق.
- لا عبور بين الشركات؛ لا أسرار في الاستجابة؛ الواجهة لا تولّد بيانات تقارير محلياً.

---

## 10. نتائج اختبار API (متوقعة)

مع **طلبات مصادَق عليها** وقاعدة فارغة:

- يجب أن تعيد النقاط أصفاراً أو `rows: []` دون خطأ 500.
- قائمة التحقق المنطقية للمسارات أعلاه متوافقة مع `reportService`.

*(اختبار يدوي موصى به: تشغيل الخادم، تسجيل الدخول، استدعاء كل مسار بـ Bearer token.)*

---

## 11. نتائج اختبار الواجهة (تحقق يدوي)

1. فتح مركز التقارير.
2. اختيار كل تقرير MVP — لا يظهر «غير مفعّل» للتقارير الفعّالة.
3. التقارير المحاسبية المتقدمة تعرض رسالة «مرحلة محاسبية متقدمة».
4. تطبيق الفلاتر يعيد تحميل البيانات.
5. تصدير Excel وطباعة للتقرير النشط.
6. فراغ احترافي عند عدم وجود بيانات.

---

## 13. قيود معروفة

1. التقارير المحاسبية الكاملة تحتاج **محرك قيود/دفتر يومية**.
2. **PDF** = طباعة المتصفح / Electron وليس توليد PDF من الخادم.
3. التقارير الضخمة جداً قد تحتاج لاحقاً تدفقاً أو تصديراً من الخادم.
4. تقرير الرواتب **ملخص MVP** وليس محاسبة رواتب كاملة.

---

## 14. خاتمة

تم تنفيذ MVP مركز التقارير وفق المواصفات: بيانات حقيقية من PostgreSQL، فلاتر فاعلة، تصدير Excel، طباعة/PDF، RTL عربي، والحفاظ على هيكل الواجهة المعتمد مع تمييز واضح للتقارير غير الجاهزة دون خداع بصري أو بيانات وهمية.
