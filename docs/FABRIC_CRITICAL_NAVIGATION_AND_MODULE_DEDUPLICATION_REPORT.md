# تقرير حرج — تنظيف التكرار في التنقل والوحدات وتوحيد المزايا داخل الأقسام المعتمدة

**المشروع:** نظام إدارة مستودعات الأقمشة (ERP)  
**نوع المهمة:** تنظيف UX حرج (CRITICAL UX CLEANUP)  
**الحالة:** ✅ مكتمل

---

## 1. الملخص التنفيذي

كان التطبيق يحتوي على عدة أقسام مكررة بتصاميم مختلفة ومسؤوليات متداخلة، مما يُعدّ غير مقبول لنظام ERP محترف ويُهدد تجربة المستخدم المعتمدة من العميل. تم تجميد العمل على المزايا الجديدة (الطباعة الصامتة في المرحلة 7 لا تزال موجودة لكن لا توجد إضافات جديدة) والتركيز على دمج الأقسام المتكررة داخل الأقسام الأساسية المعتمدة دون كسر التصميم.

**النتيجة:** **تم تنظيف التكرار بدون كسر التصميم**.

## 7. ما تم دمجه وأين

### دمج "تعريفات الأقمشة" داخل "تصنيفات الأقمشة"

`Categories.tsx` أصبحت صفحة ذات شريط تبويبات:

```
┌─────────────────────────────────────────────────────┐
│ تصنيفات الأقمشة                                      │
│ شجرة التصنيفات والخامات والألوان والمتغيرات        │
├─────────────────────────────────────────────────────┤
│ [التصنيفات] [الخامات] [الألوان] [المتغيرات]          │
├─────────────────────────────────────────────────────┤
│                                                     │
│   محتوى التبويب النشط                              │
│   (يتزامن مع ?tab= في URL)                         │
│                                                     │
└─────────────────────────────────────────────────────┘
```

- **التبويب "التصنيفات" (الافتراضي):** شجرة التصنيفات الأصلية (CategoriesTreePanel) — كل المنطق محفوظ.
- **التبويب "الخامات":** يستخدم مكوّن `ItemsTab` المُصدَّر من `FabricMasterData.tsx` — كل CRUD على الخامات يعمل كما كان.
- **التبويب "الألوان":** يستخدم مكوّن `ColorsTab` — كل CRUD على الألوان يعمل كما كان.
- **التبويب "المتغيرات":** يستخدم مكوّن `VariantsTab` — كل CRUD على المتغيرات يعمل كما كان.

التصميم الموحَّد (نفس الـ tabs strip، نفس المظهر، نفس التباعد) يحافظ على ثبات تجربة المستخدم.

### دمج "إعدادات سطح المكتب" داخل "إعدادات النظام"

`SystemSettings.tsx` أضافت بنداً في القائمة الجانبية:

```
┌─────────────────────────────────────────────────────┐
│ إعدادات النظام                                       │
├─────────────────────────────────────────────────────┤
│ ◉ بيانات المنشأة                                    │
│ ○ الإعدادات العامة                                  │
│ ○ تطبيق سطح المكتب     ← جديد، مدمج                │
│ ○ الفوترة والضرائب                                  │
│ ○ المستخدمين والصلاحيات                            │
│ ○ إعدادات المراسلة                                  │
│ ○ قواعد البيانات (النسخ الاحتياطي)                  │
│ ○ الثيمات و عرض                                     │
└─────────────────────────────────────────────────────┘
```

- التبويب يُحقن من `DesktopSettingsBody` المُصدَّر — كل المنطق محفوظ.
- التبويب يتزامن مع URL: `/settings?tab=desktop`.
- تغطي وظيفته: API URL، اختبار الاتصال، طابعة افتراضية، الطباعة الصامتة، أبعاد اللصاقة.

### إزالة بانر "استيراد Excel الحقيقي متاح الآن!"

تم حذف مكوّن `<NewImportBanner />` بالكامل من `Purchases.tsx`. الإجراء البديل: زر **"استيراد فاتورة شراء"** في الجزء العلوي من الصفحة (موجود مسبقاً) + رابط جديد "سجل استيراد فواتير الشراء" يقود إلى `/purchases/import-batches`.

## 12. نتيجة بناء Electron

`electron:compile` نجح بدون أخطاء. كل التغييرات في الواجهة والمسارات لا تتطلب تعديل عملية Electron الرئيسية. الحزم النهائية في `release/win-unpacked/` لا تتطلب إعادة بناء (لكن سيتم إعادة بناؤها تلقائياً عند تشغيل `npm run electron:pack`). جسر `window.fabricApp` و IPC channels لم تتغير.
